Ты senior QA инженер по CRM Corp.

Задача: на основе чек-листа сформировать тестовые сценарии для API создания корпоративного клиента.

Требования к ответу:
- Используй только факты из чек-листа.
- Не добавляй новые бизнес-правила, которых нет в чек-листе.
- Покрой позитивные и негативные случаи.
- Для каждого сценария укажи:
  - id (формат: SC-CRM-XXX),
  - title,
  - type (positive/negative),
  - preconditions (массив строк),
  - steps (массив строк),
  - expected_result.
- Пиши коротко и однозначно, без лишнего текста.

Формат ответа:
- Верни только JSON-объект вида:
{
  "scenarios": [
    {
      "id": "SC-CRM-001",
      "title": "...",
      "type": "positive",
      "preconditions": ["..."],
      "steps": ["..."],
      "expected_result": "..."
    }
  ]
}
- Без markdown, комментариев и пояснений.
- JSON должен начинаться с { и заканчиваться }.

CHECKLIST:
Чек-лист CRM Corp (создание корпоративного клиента):

1. Передать все корректные данные компании -> статус 201, клиент создан.

2. Использовать уже существующую пару ИНН + КПП -> статус 409, ошибка дубликата.

3. Отправить запрос без обязательных полей (externalId, inn, fullName) -> ошибка валидации.

4. Передать пустые строки или null в обязательных полях -> статус 400 Bad Request, без падения сервиса.

5. Передать некорректный формат ИНН/КПП (длина, символы) -> ошибка валидации.

6. Передать неразрешенное значение channel/source -> ошибка валидации.

7. Передать некорректную дату регистрации или дату из будущего -> запрос отклоняется.

8. Передать спецсимволы в наименовании организации (дефис, кавычки, апостроф) -> данные сохраняются корректно.
