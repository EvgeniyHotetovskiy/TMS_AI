Ты Senior QA Engineer.

Контекст: CRM Corp, процесс создания корпоративного клиента.

На основе артефактов:
- исходный чек-лист,
- сгенерированные test cases,
- AI code review,

выяви дефекты, несоответствия требованиям, пропущенные проверки и риски.

Критерии поиска:
- конфликт между чек-листом и test cases;
- неполное покрытие обязательных негативных сценариев;
- недостоверные expected result;
- риск ложноположительных/ложноотрицательных проверок в автотестах;
- технические риски, влияющие на качество тестирования.

Формат ответа:
- Верни ТОЛЬКО JSON.
- Если проблем нет:
{ "status": "NO_BUGS_FOUND" }

- Если проблемы есть, верни:
{
  "status": "BUGS_FOUND",
  "bugs": [
    {
      "id": "BUG-001",
      "title": "",
      "severity": "blocker|critical|major|minor",
      "priority": "high|medium|low",
      "preconditions": "",
      "steps_to_reproduce": [],
      "actual_result": "",
      "expected_result": "",
      "probable_root_cause": "",
      "evidence": ""
    }
  ]
}

Правила:
- Только валидный JSON.
- Без markdown.
- Без пояснений вне JSON.
- Не придумывай факты, которых нет в артефактах.

CHECKLIST:
{{CHECKLIST}}

TESTCASES:
{{TESTCASES}}

CODE REVIEW:
{{REVIEW}}
